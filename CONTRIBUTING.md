# Contribution Guide

## Workflow Git
1. Créer une branche pour chaque tâche :
   - `docs/<sujet>` pour documentation
   - `feat/<sujet>` pour fonctionnalités
   - `fix/<sujet>` pour corrections

2. Ouvrir une Pull Request (PR) pour toute modification.
3. Relecture obligatoire par au moins 1 membre de l’équipe.

## Conventional Commits
Utiliser :
- `docs: ...` (documentation)
- `feat: ...` (feature)
- `fix: ...` (bugfix)
- `chore: ...` (maintenance)
- `refactor: ...` (refactor)

Exemples :
- `docs: add project vision and scope`
- `docs: add requirements list`
- `feat: create patient tracking state machine`

## Definition of Done
- Document/Code terminé
- Relecture faite
- Versionné dans Git
- Pas d’informations contradictoires
